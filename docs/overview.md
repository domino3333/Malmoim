# 말모임 Overview

## 1. 개요

말모임은 세미나, 스터디, 발표 환경에서 참가자들이 질문을 올리고 진행자가 질문을 실시간으로 관리할 수 있는 Q&A 웹이다.

1차 버전은 발표 현장에서 가장 자주 쓰이는 실시간 Q&A 흐름에 집중한다. 참가자는 별도의 복잡한 절차 없이 방에 입장해 질문을 작성하고, 다른 참가자의 질문에 추천을 누를 수 있다. 진행자는 올라온 질문을 확인하고 답변 완료 또는 숨김 처리하면서 발표 흐름을 관리한다.

## 2. 프로젝트 목적

말모임의 목적은 발표자와 참가자 사이의 질문 흐름을 더 편하게 만드는 것이다.

오프라인 발표나 스터디에서는 질문하기가 부담스럽거나, 질문이 한 번에 몰리면 진행자가 관리하기 어렵다. 말모임은 이런 상황에서 참가자가 질문을 쉽게 남기고, 진행자가 중요한 질문부터 확인할 수 있게 돕는다.

## 3. 핵심 문제

- 참가자는 발표 중 손을 들고 질문하기 부담스러울 수 있다.
- 진행자는 어떤 질문이 많은 사람에게 중요한지 알기 어렵다.
- 채팅방 방식은 대화가 빠르게 흘러가 질문 관리에 적합하지 않을 수 있다.
- 발표가 끝난 뒤 어떤 질문이 나왔는지 정리하기 어렵다.

## 4. 대상 사용자

### 4.1 진행자

세미나, 스터디, 강의, 발표를 진행하는 사람이다. Q&A 방을 만들고 참가자에게 입장 코드를 공유한다. 질문 목록을 확인하며 답변할 질문을 고르고, 답변 완료 또는 숨김 상태로 관리한다.

### 4.2 참가자

세미나, 스터디, 강의, 발표에 참여하는 사람이다. 방 코드나 링크로 Q&A 방에 입장하고, 질문을 작성하거나 다른 질문에 추천을 누른다.

## 5. 1차 MVP 범위

1차 MVP는 실시간 Q&A 경험을 완성하는 데 집중한다.

포함할 기능은 다음과 같다.

- 진행자 회원가입 및 로그인
- Q&A 방 생성
- 방 코드 또는 링크를 통한 참가자 입장
- 익명 질문 작성
- 질문 목록 조회
- 질문 추천
- 추천순/최신순 정렬
- 진행자의 질문 답변 완료 처리
- 진행자의 질문 숨김 처리
- 질문 등록, 추천, 상태 변경의 실시간 반영
- 발표용 질문 목록 화면

## 6. 1차 MVP에서 제외할 기능

다음 기능은 추후 확장 대상으로 둔다.

- 투표 기능
- 퀴즈 기능
- 설문 기능
- 참가자 로그인
- 결제 기능
- 팀/조직 기능
- AI 질문 요약
- 발표 자료 연동
- 복잡한 통계 대시보드

## 7. 서비스 방식

말모임의 Q&A는 일반 채팅방이 아니라 실시간 질문 게시판 방식이다.

참가자가 질문을 올리면 같은 방에 들어와 있는 참가자 화면, 진행자 화면, 발표용 화면에 질문이 즉시 반영된다. 다른 참가자가 추천을 누르면 추천 수가 실시간으로 바뀐다. 진행자가 질문을 숨기거나 답변 완료 처리하면 그 상태도 즉시 화면에 반영된다.

## 8. 기술 스택

- Frontend: React
- Backend: Spring Boot
- Database: MySQL
- Realtime: WebSocket 또는 STOMP 기반 실시간 이벤트 처리
- Deployment: 추후 배포 환경 선정

## 9. 성공 기준

말모임 1차 버전의 성공 기준

> 진행자가 Q&A 방을 만들고, 참가자가 질문을 올리면, 모든 관련 화면에 질문과 상태 변화가 실시간으로 반영된다.

기능을 많이 넣는 것보다 이 핵심 흐름이 안정적으로 동작하는 것을 우선한다.

## 10. 향후 확장 방향

1차 버전 이후에는 다음 방향으로 확장할 수 있다.

- 행사별 Q&A 기록 관리
- 질문 검색 및 필터링
- 질문 내보내기
- 투표/설문/퀴즈 기능
- QR 코드 입장
- AI 기반 질문 묶음 및 요약
- 진행자 통계 화면
